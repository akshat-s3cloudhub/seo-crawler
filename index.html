📌 Project Overview

A real-time SaaS-style SEO & AEO (Answer Engine Optimization) content audit dashboard built for WNS.com on Squadbase. It audits 832 live content pages, scores them across 6 categories (60 points total), and provides live monitoring, change detection, alerts, and deep-dive inspection — all auto-refreshing every 30 seconds without any manual page reload.

Table 3: wns_seo_change_log — Change Detection Log

CREATE TABLE wns_seo_change_log (
  id              SERIAL PRIMARY KEY,
  url             TEXT NOT NULL,
  detected_at     TIMESTAMP NOT NULL DEFAULT NOW(),
  change_type     TEXT NOT NULL,
  -- Values: seo_improvement | seo_regression | band_change | new_page | page_removed

  new_value       TEXT,

CREATE INDEX idx_change_log_detected_at ON wns_seo_change_log(detected_at DESC);

seo-aeo-kpi-summary No params   Cache: 300s

Purpose: Top-level KPI metrics for the dashboard header — total pages, average score, AI-ready %, critical pages count.

SELECT
    COUNT(*) AS total_pages,
    ROUND(AVG(total_score), 1) AS avg_total_score,
    ROUND(AVG(total_score) * 100.0 / 60, 1) AS avg_score_pct,
    ROUND(AVG(cat_a_score), 1) AS avg_cat_a,
    ROUND(AVG(cat_b_score), 1) AS avg_cat_b,
    ROUND(AVG(cat_c_score), 1) AS avg_cat_c,
    ROUND(AVG(cat_d_score), 1) AS avg_cat_d,
    ROUND(AVG(cat_e_score), 1) AS avg_cat_e,
    ROUND(AVG(cat_f_score), 1) AS avg_cat_f,
    SUM(CASE WHEN band IN ('Elite','Strong') THEN 1 ELSE 0 END) AS ai_ready_pages,
    ROUND(SUM(CASE WHEN band IN ('Elite','Strong') THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) AS ai_ready_pct,
    SUM(CASE WHEN band = 'Critical' AND total_score > 0 THEN 1 ELSE 0 END) AS critical_pages,
    ROUND(SUM(CASE WHEN band = 'Critical' AND total_score > 0 THEN 1 ELSE 0 END) * 100.0 / COUNT(*), 1) AS critical_pct
FROM wns_seo_content_assets
WHERE crawler_status != 'removed'
seo-aeo-asset-scorecard asset_type (default: all)band (default: all)   Cache: 120s

seo-aeo-score-band-distribution No params   Cache: 300s

Purpose: Count and percentage of pages per band (Elite → Critical). Powers the band distribution chart.

SELECT
    band,
    COUNT(*) AS page_count,
    ROUND(COUNT(*) * 100.0 / SUM(COUNT(*)) OVER (), 1) AS pct_of_total,
    ROUND(AVG(total_score), 1) AS avg_total_score,
    MIN(total_score) AS min_score,
    MAX(total_score) AS max_score,
    priority_action
FROM wns_seo_content_assets
WHERE crawler_status != 'removed' AND total_score > 0
GROUP BY band, priority_action
ORDER BY
    CASE band
        WHEN 'Elite'    THEN 1
        WHEN 'Strong'   THEN 2
        WHEN 'Average'  THEN 3
        WHEN 'Weak'     THEN 4
        WHEN 'Critical' THEN 5
        ELSE 6
    END
seo-aeo-scores-by-asset-type No params   Cache: 300s

Purpose: Average total score and A–F category scores broken down by content type (Article, Blog, Case Study, etc.).

SELECT
    asset_type,
    COUNT(*) AS page_count,

    ROUND(AVG(cat_e_score), 1) AS avg_cat_e,
    ROUND(AVG(cat_f_score), 1) AS avg_cat_f,

    SUM(CASE WHEN band = 'Strong'   THEN 1 ELSE 0 END) AS strong_count,
    SUM(CASE WHEN band = 'Average'  THEN 1 ELSE 0 END) AS average_count,
    SUM(CASE WHEN band = 'Weak'     THEN 1 ELSE 0 END) AS weak_count,
    SUM(CASE WHEN band = 'Critical' THEN 1 ELSE 0 END) AS critical_count

seo-aeo-scores-by-industry No params   Cache: 300s

Purpose: Scores broken down by industry vertical. Shows which sectors need the most SEO/AEO attention.

SELECT
    COALESCE(industry, 'Unknown') AS industry,
    COUNT(*) AS page_count,
    ROUND(AVG(total_score), 1) AS avg_total_score,
    SUM(CASE WHEN band = 'Elite'    THEN 1 ELSE 0 END) AS elite_count,
    SUM(CASE WHEN band = 'Strong'   THEN 1 ELSE 0 END) AS strong_count,
    SUM(CASE WHEN band = 'Average'  THEN 1 ELSE 0 END) AS average_count,
    SUM(CASE WHEN band = 'Weak'     THEN 1 ELSE 0 END) AS weak_count,
    SUM(CASE WHEN band = 'Critical' THEN 1 ELSE 0 END) AS critical_count
FROM wns_seo_content_assets
GROUP BY industry
HAVING COUNT(*) >= 3
ORDER BY page_count DESC
LIMIT 30

crawler-stats No params   Cache: 15s

Purpose: Overall crawler status — total/crawled/pending/failed counts + last crawl time. Powers the Crawler Control Center status panel.

SELECT
    COUNT(*) AS total_urls,
    COUNT(CASE WHEN crawler_status = 'crawled' THEN 1 END) AS crawled_count,
    COUNT(CASE WHEN crawler_status = 'pending' OR crawler_status IS NULL THEN 1 END) AS pending_count,
    COUNT(CASE WHEN crawler_status = 'failed' THEN 1 END) AS failed_count,
    COUNT(CASE WHEN total_score > 0 THEN 1 END) AS scored_count,
    MAX(last_crawled_at) AS last_crawled_at,
    ROUND(COUNT(CASE WHEN crawler_status = 'crawled' THEN 1 END) * 100.0 / NULLIF(COUNT(*), 0), 1) AS crawl_pct
FROM wns_seo_content_assets
crawler-jobs No params   Cache: 10s

Purpose: Recent 20 crawl jobs with status, progress, and timing. Powers the job history table and active job progress bar.

SELECT
    id, job_type, status, started_at, completed_at,
    total_urls, crawled_urls, failed_urls, changed_urls, triggered_by,
    ROUND(EXTRACT(EPOCH FROM (COALESCE(completed_at, NOW()) - started_at)) / 60, 1) AS duration_minutes
FROM wns_seo_crawl_jobs
ORDER BY started_at DESC NULLS LAST

LIMIT 50

🕷️ Crawler System

GitHub Repository: https://github.com/akshat-s3cloudhub/seo-crawler
Clone this to run or deploy the crawler on the client's infrastructure.

Tech Stack

LibraryPurpose
Node.js + TypeScriptCrawler runtime
PlaywrightHeadless browser — handles JS-rendered pages
CheerioHTML parser — extracts SEO/AEO parameters
pg (node-postgres)PostgreSQL connection for writing scores
ts-nodeTypeScript execution without compile step

What the Crawler Does (Per Page)

  1. Fetches the page URL using Playwright (handles JavaScript rendering, SPAs)
  2. Passes the rendered HTML to Cheerio for parsing
  3. Extracts all 30 SEO/AEO parameters (A1–F3) from the HTML structure
  4. Applies the 60-point scoring rubric to calculate category scores (A–F)
  5. Classifies the page into a band (Elite / Strong / Average / Weak / Critical)
  6. Upserts the row in wns_seo_content_assets (update if exists, insert if new)
  7. Saves a timestamped snapshot to wns_seo_crawl_history
  8. Compares old vs new score — logs any change to wns_seo_change_log
  9. Updates the crawl job progress in wns_seo_crawl_jobs

# Clone the repo
git clone https://github.com/akshat-s3cloudhub/seo-crawler
cd seo-crawler

# Install dependencies

# Set your database URL
export DATABASE_URL="postgresql://user:password@host:5432/dbname"

# Run crawler (crawls all 832 pages, takes ~8 minutes)
npx ts-node src/crawler.ts

Deploying to Railway (Production)

  1. Go to railway.app → Create New Project → Deploy from GitHub Repo
  2. Select the seo-crawler repository
  3. In Railway settings, add environment variable: DATABASE_URL = your PostgreSQL connection string
  4. In Railway Cron Jobs, create two schedules:

    Full Crawl: 0 2 * * 0 (Every Sunday at 2:00 AM)
    Incremental Crawl: 0 3 * * * (Every day at 3:00 AM)
  5. Deploy — Railway will auto-deploy on every git push to main

  1. Go to squadbase.com and sign up for an account

  4. Note down the connection string — you'll need it for the crawler and DB import

Step 2 — Export Database from Current Project (You Do This)

# Option A: Using pg_dump (recommended — full export)
pg_dump \
  -h YOUR_CURRENT_HOST \
  -U YOUR_CURRENT_USER \
  -d YOUR_CURRENT_DB_NAME \
  --no-owner \
  --no-acl \
  -F p \
  -f wns_seo_full_export.sql

# This creates a single .sql file containing:
# - All 5 CREATE TABLE statements
# - All 832 rows from wns_seo_content_assets
# - All crawl history, change logs, crawl jobs
# - All scoring logic reference data
If you don't have direct DB access, contact Squadbase support to request a database export from your current project.

Step 3 — Import Database into Client's Squadbase DB (Client Does This)

# Import the SQL dump into the client's new database
psql \
  -h CLIENT_DB_HOST \
  -U CLIENT_DB_USER \
  -d CLIENT_DB_NAME \
  -f wns_seo_full_export.sql

# This recreates all 5 tables and imports all data instantly
